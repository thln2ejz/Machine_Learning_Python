Under the probabilistic approach to classification, an object $x$ is classified as belonging to class $\omega_i$ with the highest posterior probability $p(\omega_i|x)$. For the majority of classifiers that follow this approach, the posterior probability is derived using the class-conditional density $p(x|\omega_i)$ in conjunction with Bayes' theorem. In the first two exercises however, we investigate one classifier that is an exception to this rule. Although its classification decision is still based on maximum posterior probability, the logistic classifier attempts to directly model the posterior distributions without first estimating the class-conditional densities.

Basics of Classification¶

Normal-based Discriminants¶

Following the probabilistic approach to classification, if the conditional density function is known, everything about the classification problem is known, and an optimal decision may be reached resulting in the lowest possible error in the theoretical case: Bayes' error. However, since in practice we need to estimate the class conditional densities, the final classification rate will depend on how accurate our estimate really is. Density estimation is not a trivial task. Some methods assume a Gaussian model for the class densities, whereas others may use the mixture of Gaussians method over the entire feature space or make use of Parzen density estimation or k-nearest neighbor density estimation. In the following exercise, we will model the class density functions with a Gaussian model. Under the probabilistic approach to classification, the fundamental principles for solving any classification problem involve the following steps:

  • Split your dataset into a training set and a test set.
  • Estimate the class conditional probability density function for every class using the training set.
  • For every object in the test set, compute the class conditional density.
  • For every object in the test set, compute the posterior probability using Bayes' theorem.
  • Assign every test object to the class with the highest posterior probability. The object will carry the corresponding class label.

The Curse of Dimensionality¶

The curse of dimensionality is a very important concept in machine learning. It sets a limit on the number of features to use for a given classification problem. As the dimension of the feature space increases, the classification error over an independent test set begins to increase as well after some point. In the next exercise, we shall explore this phenomenon more closely.

We vary the number of dimensions as follows: $d = [2, 3, 5, 10, 25, 100]$, and for each case, we generate 10 random draws from two $d$-dimensional Gaussian distributions to create training sets and test sets, then we average the classification results. dimensionality = [2,3,5,10,25,100]
error_test_mean, error_test_std = [],[]

for i in dimensionality:
    error_test = [] 
    for j in range(10):
        X1 = np.random.multivariate_normal(mean=np.zeros(i),cov=np.identity(i),size=10+1000) 
        X2 = np.random.multivariate_normal(mean=np.append(2,np.zeros(i-1)),cov=np.identity(i),size=10+1000)

        X_train = np.concatenate([X1[:10,:],X2[:10,:]],axis=0)
        X_test = np.concatenate([X1[10:,:],X2[10:,:]],axis=0)
        y_train = np.repeat([1,2],10)
        y_test = np.repeat([1,2],1000)
        
        model = KNeighborsClassifier(n_neighbors=1).fit(X_train, y_train)
        y_pred = model.predict(X_test)
        error_test.append((y_pred!=y_test).mean())
        
    error_test_mean.append(np.mean(error_test))
    error_test_std.append(np.std(error_test))

plt.errorbar(dimensionality, error_test_mean, error_test_std)
plt.scatter(dimensionality, error_test_mean)
plt.xlabel('dimensionality')
plt.ylabel('test error')
plt.show()
