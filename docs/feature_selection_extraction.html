Feature Selection & Extraction¶

J.A.¶

You have seen in the previous lab that visualizing datasets in 2 or 3 dimensions is quite convenient. But when the number of features is in the order of tens, hundreds, or thousands, feature reduction becomes an essential tool. Visualization is one motivation for feature reduction. Another reason for reducing dimensionality is to improve statistical estimates such as when estimating the covariance matrix of a dataset. Sometimes a large amount of data is required to estimate even simple parameters in 3 dimensional spaces. With higher dimensions, the number of samples required grows exponentially, wherefore it would be far easier to remove dimensions rather than add samples, especially that this latter may not be possible. Feature reduction is crucial for a large portion of classification tasks as well as regression problems. In this lab, we explore two general approaches to dimensionality reduction: feature selection and feature extraction.

Feature Selection¶

In feature selection, we simply select a subset of features from the available ones and drop the others. In order to do this, we require a criterion function to assess a given feature subset and a method to choose subsets, that is, we require a search algorithm. A number of search algorithms were presented, namely 'the best individual d features', forward selection, backward selection, plus-l-take-away-r selection, and branch & bound. Scikit-learn provides RFE() that performs backward feature selection.

import numpy as np
import pandas as pd
from sklearn import datasets
from sklearn.feature_selection import RFE
from sklearn.svm import SVC

iris = datasets.load_iris()
meas = iris.data        
y = iris.target         
X = np.random.multivariate_normal(mean=[0]*10, cov=np.diag([1]*10), size=150)
X = pd.DataFrame(X, columns=[f'V{i}' for i in range(10)])

X[['V0','V2','V4','V6']] = meas #or X.iloc[:,[0,2,4,6]]

svc = SVC(kernel="linear", C=1)
rfe = RFE(estimator=svc, n_features_to_select=4, step=1)
rfe.fit(X, y)
rfe.support_.nonzero()

Exercise 10. The error function contains local mimina all around the global minimum at (0,0). This makes it impossible for gradient descent algorithms to converge to the global minimum unless the initial starting point happens to fall very close to the global minimum which is an unrealistic assumption. Instead, we will use a genetic algorithm in an attempt to find the global minimum efficiently. Use geneticalgorithm() from the source shown below: the first argument is a call to the error function; also specify the variable boundaries which in this case are $[-3,3]$. You may also set the number of individuals in the population using population_size, the ratio of elite individuals passing through to the next generation without any change using elit_ratio, and the fraction of offspring that is generated by crossover using crossover_probability.

from geneticalgorithm import geneticalgorithm as ga  #https://pypi.org/project/geneticalgorithm/

varbound = np.array([[-3,3]]*2)
algorithm_param = {'max_num_iteration':1000,
                   'population_size':20,
                   'mutation_probability':0.2,
                   'elit_ratio':2/100,
                   'crossover_probability':0.8,
                   'parents_portion': 0.3,
                   'crossover_type':'uniform',
                   'max_iteration_without_improv':None}

model = ga(function=Rastrigin, dimension=2, variable_type='real', 
           variable_boundaries=varbound, algorithm_parameters=algorithm_param)
model.run()
