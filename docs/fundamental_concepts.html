Histogram¶

In the probabilistic approach to classification, knowledge of density is all that is required to solve a classification problem. Knowledge of the true density is however rarely given, and one has to estimate the probability density function in feature space. The simplest way to visualize the probability density function in one or two dimensions is by constructing a histogram. The problem with histograms is choosing a suitable number of bins, wherefore if the number of bins is very small, useful information will be hidden, whereas if the number of bins is very large, then each bin may not contain enough samples for the histogram to be meaningful.

Box Plot¶

A boxplot provides a way for visualizing data symbolically by summarizing a histogram plot. plt.boxplot() may be used for this purpose. A boxplot consists of a box with lines extended outward, the whiskers. The position of the box, whiskers, and other values are based on rank statistics obtained by the following procedure:

  1. Data samples are sorted in ascending order.

  2. The median is obtained; this is the 50th percentile.

  3. The $r^{th}$ percentile is defined as the value at $(r/100) \times n$, where $n$ is the number of samples in the data.

The horizontal line inside the box (not necessarily at the center) represents the median value. The lower and upper boundaries of the box are the 25th and 75th percentile values, whereby 25% of the data falls below the box's lower boundary and 25% falls above the upper box boundary. The inter-quartile range is the difference between these two percentile values. The whiskers are defined either at a distance of 1.5 times the inter-quartile range from the median or at the distance to the furthest sample from the median, whichever of the two is smaller. All samples falling outside the whiskers are shown separately with round circles; these are outliers.

Central Limit Theorem¶

The normal or Gaussian distribution is widely used and is found in many statistical distributions in nature. The central limit theorem provides an explanation for this fact. It states that the mean (or sum) of a large number of identically and independently distributed samples taken randomly from any distribution with mean $u$ and variance $v$ will follow approximately a Gaussian distribution. As the number of samples $n$ increases, the distribution of the mean approaches exactly the Gaussian distribution with mean $u$ and variance $v/n$. Since measured quantities in the real world are usually a sum of numerous unobserved random quantities coming from similar distributions, the central limit theorem helps explain why the measured quantity usually ends up following a normal distribution.

Parameter Estimation¶

In data analytics problems, we often employ training sets and test sets especially for assessing classification. It is often not acceptable to report a value based on a single experiment. In general, when estimating a parameter or reporting a value in any kind of report it should be standard practice to repeat an experiment if possible a number of times to obtain a fair estimate and an overview of the spread of the value estimated. In what follows we shall illustrate this concept by estimating the mean and variance of a predefined dataset.

A bandwidth of 1 (default) is used in the plot below.

Try different bandwidth values for the kernel. What happens with lower values and higher values. Which kernel width do you think is optimal for this dataset?

To evaluate the fit of a density model to the actual data, an error measure is required. In this case, we may use the log-likelihood defined as:

$LL(x) = ln \left( \prod_i \widehat{p}(x_i) \right) = \sum_i ln(\widehat{p}(x_i))$

If the model fits well, then the log-likelihood should be high. Different kernel widths $h$ result in different models. Thus, using the log-likelihood, we can choose an optimal kernel width $h$ that corresponds to a maximum log-likelihood measure. Note that the log-likelihood must be evaluated on an independent test set; that is, it is not acceptable to evaluate the log-likelihood using the same training set on which the probability density function was fitted which would lead to a biased (optimistic) evaluation.
